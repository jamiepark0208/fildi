## 2026-06-14 — Session Summary: FactSet Proxy Activation

### Root cause of initial failures
1. PM2 was started via `pm2 start proxy.js` (inline shell env vars) not `ecosystem.config.js`, so env vars didn't survive PM2 restarts → crash loop "Missing env vars"
2. FactSet Fundamentals API v2 returns 403 — the developer account (`USCMARSHALL-2393811`) only subscribes to the **Overview Report Builder API**, not Fundamentals/Prices/Entity APIs

### What was done
- Diagnosed PM2 crash loop from SSH logs (`pm2 logs factset-proxy`)
- Fixed: `pm2 delete factset-proxy && pm2 start ecosystem.config.js && pm2 save` (env vars now persist across reboots)
- Discovered correct available endpoint via FactSet error report XLSX + GitHub SDK docs: `GET /report/overview/v1/financial-highlights?id=TICKER-US`
- Rewrote `fetchFactSet()` in `stock-data-manager.ts` to use Overview Report Builder API instead of Fundamentals v2
- Fixed STACH 2.0 scale parsing bug: FactSet omits `scale` key on some FIN rows → code now defaults to scale=6 (millions) for `metric=FIN` non-percentage rows
- Rebuilt api-server (`node build.mjs` in `artifacts/api-server/`)
- End-to-end test confirmed: `POST /api/sdm/refresh/NVDA?admin=true` → `{"source":"factset","dataQualityScore":"0.5667"}`

### Data quality: FactSet vs prior sources
| Source | Quality Score | Fields |
|---|---|---|
| EDGAR | 0.27 | Revenue, Net Income, Cash, Equity (sparse) |
| FactSet Overview | 0.57 | + Margins, ROE/ROA, EPS, FCF, D/E, Cash Flow |
| FMP (complementary) | ~0.8+ | Fills PE, P/B, ROIC, Beta, WACC, Shares |

### Key lesson: PM2 + env vars
Always start PM2 via `ecosystem.config.js` (not inline), then `pm2 save`. Shell env vars are NOT persisted in PM2 dump. If you use inline env, every PM2 restart (crash, reboot) drops them.
